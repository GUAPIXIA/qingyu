# 基于文档创建/完善项目

## 使用方式

```
/implement-from-docs @文档路径.md
```

## 工作流程

### 1. 读取并分析文档

- 读取用户指定的文档文件
- 识别文档中的：
  - 项目概述和目标
  - 功能需求列表
  - 技术栈要求
  - 数据库设计
  - API 接口定义
  - 页面/组件需求

### 2. 检查现有项目状态

- 扫描项目目录结构
- 检查已有的配置文件
- 识别已实现的功能
- 确定需要新增或修改的部分

### 3. 制定实施计划

创建 TODO 列表，按优先级排序：
1. 项目配置 (pom.xml, package.json, 配置文件)
2. 数据库/模型层
3. 后端 API (Controller → Service → Mapper)
4. 前端页面和组件
5. 集成测试

### 4. 逐步实现

按照计划逐项实现：
- 每完成一个功能点，更新 TODO 状态
- 遇到问题时及时报告用户
- 保持代码风格一致

### 5. 验证和总结

- 运行项目确认功能正常
- 总结已实现的功能
- 列出可能的后续优化点

## 注意事项

- 优先实现核心功能，再处理边缘情况
- 保持与现有代码风格一致
- 遇到不确定的需求，及时询问用户
- 定期保存进度，避免丢失工作
