# 轻 Tavern 全功能补全计划（对标 Tavo）

## 概述

用户要求：1) 修复 PNG 导入失败和封面不显示；2) 添加 {user} 用户人设；3) TTS 暂停功能；4) 对比 Tavo 缺失的核心功能全部补上。经全面探索代码库后，制定以下完整计划。

## 当前状态分析

### 已有功能
角色管理、对话交互、4 种 AI 后端、世界书、预设、群聊、TTS（仅停止）、图片发送、主题系统、帮助页

#### 1.2 修复 iTXt chunk 解析

**文件：** `electron/services/charCard.ts` 第 50-59 行
- iTXt 格式：`keyword\0  flag(1)  method(1)  language\0  translated\0  text`
- 修复：从 `nullIdx + 3` 开始找下一个 `\0`（language tag 结束），再找下一个 `\0`（translated keyword 结束），之后即为 text
- 代码：
  ```typescript
  // nullIdx+1 = compression flag, nullIdx+2 = compression method
  let pos = nullIdx + 3
  const langEnd = data.indexOf(0, pos)  // language tag 的 \0
  if (langEnd < 0) continue
  pos = langEnd + 1
  const transEnd = data.indexOf(0, pos)  // translated keyword 的 \0
  if (transEnd < 0) continue
  const textStart = transEnd + 1
  chunks[key] = data.toString('utf-8', textStart)
  ```

#### 1.3 优化封面显示

**文件：** `src/components/character/CharacterCard.tsx`
- `<img>` 加 `onError` 回调，失败时设置 `imgError=true`，显示首字母 fallback

**文件：** `src/components/chat/MessageBubble.tsx`
- 对话中的角色头像 `<img>` 同样加 `onError` 处理

## 假设与决策

1. **不自动打包** - 按用户要求，修复后等用户测试确认再打包
2. **HTML 渲染默认关闭** - 安全考虑，需用户在设置中手动开启
3. **翻译复用现有 AI 适配器** - 不接入第三方翻译 API，用当前配置的模型翻译
4. **多会话存储** - 路径改为 `chats/{characterId}/{sessionId}.jsonl`，兼容旧数据（迁移旧 `messages.jsonl` 为默认会话）
5. **正则作用域** - 输入正则在发送前应用，输出正则在 AI 回复后应用
6. **状态栏解析** - 从 AI 回复中用正则 `[Status: key=value]` 提取状态更新

## 验证步骤

1. **PNG 导入** - 导入无效 PNG 显示错误；导入有效卡显示封面
2. **变量替换** - 设置用户名后，对话中 `{{user}}` 被替换
3. **TTS** - 朗读 -> 暂停 -> 恢复 -> 停止
4. **多会话** - 新建对话 -> 切换 -> 删除
5. **快捷设置** - 聊天页右侧面板切换预设/世界书
6. **上下文查看** - 弹窗显示完整 prompt
7. **HTML 渲染** - 开启后含 HTML 的消息正确渲染
8. **正则** - 创建规则后输入/输出被正确替换
9. **状态栏** - AI 回复含状态标记时状态栏更新
10. **Token 计数** - 每条消息显示 token 数
11. **对话分支** - 从中间消息分叉出新对话
12. **翻译** - 点击翻译按钮显示译文
13. **类型检查** - `npx tsc -b --noEmit` 零错误
14. **运行测试** - `pnpm electron:dev` 无运行时错误
