# 八项优化与新增功能计划

## 概述

用户提出 8 项请求：1) 批量导入多个角色卡并显示结果统计；2) 优化角色编辑页面文本框高度；3) 优化角色卡封面展示效果（含点击放大预览）；4) 添加长记忆功能（可选开关、手动/自动总结、统计信息）；5) 角色卡内嵌世界书自动导入；6) 确认 V1/V2/V3 适配情况；7) 角色卡展示页面双模式切换（网格/列表）；8) 用户身份自定义头像。

---

## 当前状态分析

| # | 需求 | 当前状态 |
|---|------|----------|
| 1 | 批量导入角色卡 | 导入对话框使用 `properties: ['openFile']`（单选），只支持一次导入一个文件 |
| 2 | 编辑页文本框高度 | description `min-h-[80px]`，firstMessage `min-h-[80px]`，对长文本不够友好 |
| 3 | 角色卡封面展示 | CharacterCard 使用 `aspect-[4/3]` + `object-cover`，头像区域比例固定且裁剪图片 |
| 4 | 长记忆功能 | 完全不存在。仅 ChatInput 有一个"总结"快捷按钮（纯预制文本），无自动总结/记忆注入机制 |
| 5 | 世界书自动导入 | `normalizeCharacter` 读取了 `data.character_book?.id` 存入 `lorebookId`，但嵌入的 `character_book.entries` 数据**被丢弃**，不会自动创建世界书文件 |
| 6 | V1/V2/V3 适配 | V1/V2 已通过 `parsed.data ?? parsed` 模式适配。V3 同为 data 包装格式，理论上兼容，但缺少 `alternate_greetings`、`creator_notes`、`character_version` 等 V2+ 字段支持 |
| 7 | 角色卡双展示模式 | CharactersPage 仅支持 grid 网格布局，无列表模式；无视图切换控件 |
| 8 | 用户身份头像 | Persona 类型已有 `avatar` 字段（base64），显示时也渲染头像区；但 PersonasPage 编辑弹窗中缺少头像上传/选择控件 |

### 角色卡版本兼容实际状况

**V1（原始格式）：** 顶层 flat JSON，字段为 `name`, `description`, `first_mes`, `mes_example` 等。处理：`parsed.data ?? parsed` -> 回退到 parsed 本身。✅ 完全支持。

**V2（当前标准）：** `{ spec: "chara_card_v2", spec_version: "2.0", data: { ... } }`。导入通过 `parsed.data` 读取。导出始终输出 V2 格式。✅ 完全支持。

**V3：** 结构与 V2 相同（`data` 包装），新增可选字段如 `alternate_greetings`（备用问候语）、`creator_notes`、`character_version`。`normalizeCharacter` 已通过 `parsed.data ?? parsed` 模式兼容，核心字段（name, description, first_mes, mes_example 等）均可正常读取。⚠️ 不支持备用问候语等 V3 特有字段。

### 修改 3：优化角色卡封面展示

**当前问题：**
- CharacterCard 头像区固定 `aspect-[4/3]`，大约 4:3 比例
- 使用 `object-cover` 裁剪，长横幅或竖长图会被裁掉大部分
- 角色卡封面通常是完整画作，裁剪会损失视觉信息

**修改内容：**

**文件：`src/components/character/CharacterCard.tsx`**

方案 A（推荐）：自适应比例 + 完整展示
- 移除固定的 `aspect-[4/3]`，改为 `aspect-[3/4]`（竖版卡片比例，更符合角色卡通常的竖版画风）
- 保持 `object-cover`，但使用更接近竖版卡片的比例
- 无头像时首字母占位符保持相同比例

方案 B：完整展示 + 滚动
- 使用 `object-contain` 替代 `object-cover`，完整显示图片不裁剪
- 设置最大高度 `max-h-48` 防止卡片过高
- 背景色填充空白区域

**选择方案 A**（竖版 3:4 比例 + `object-cover`），理由：
- 角色卡封面绝大多数是竖版画作
- 3:4 比例接近常见角色立绘比例
- 不会导致卡片高度过于不一致

同时添加点击图片放大预览功能（使用已有的 Modal 组件或简单的 overlay）。

**文件：`src/components/character/CharacterCard.tsx`**
- `aspect-[4/3]` -> `aspect-[3/4]`
- 新增 `showPreview` 状态和点击图片弹出大图预览
- 预览使用 fixed overlay + `object-contain`，支持点击关闭

### 修改 5：角色卡内嵌世界书自动导入

**当前问题：**
- `normalizeCharacter` 仅提取 `data.character_book?.id` 赋给 `lorebookId`
- 嵌入的 `character_book.entries`（世界书条目数据）被完全丢弃
- 用户需要手动导出/导入世界书

**目标：** 导入角色卡时，如果包含 `character_book`，自动将其提取并保存为独立的世界书文件，更新角色的 `lorebookId`。

**修改内容：**

**文件：`electron/services/charCard.ts` - `normalizeCharacter` 函数**

在正常化流程末尾，处理 `character_book`：
```typescript
// 从角色卡数据中提取内嵌世界书
const charBook = data.character_book
if (charBook && charBook.entries && charBook.entries.length > 0) {
  const { nanoid } = require('nanoid')
  const lorebookId = nanoid()
  const lorebook: Lorebook = {
    id: lorebookId,
    name: charBook.name || `${data.name ?? parsed.name}的世界书`,
    description: charBook.description ?? '',
    entries: charBook.entries.map((e: any, i: number) => ({
      id: e.uid?.toString() ?? nanoid(),
      keywords: Array.isArray(e.key) ? e.key : (e.key ? e.key.split(',') : []),
      content: e.content ?? '',
      position: e.position === 'before' ? 'before_char' : e.position === 'after' ? 'after_char' : 'at_end',
      order: e.order ?? i,
      probability: e.probability ?? 100,
      enabled: e.disable ? !e.disable : (e.enabled ?? true),
    })),
    enabled: true,
    scanDepth: charBook.scan_depth ?? 4,
  }
  // 保存世界书文件
  const { join } = require('node:path')
  const { writeFileSync, mkdirSync } = require('node:fs')
  const lorebookDir = DIRS.lorebooks()
  mkdirSync(lorebookDir, { recursive: true })
  writeFileSync(join(lorebookDir, `${lorebookId}.json`), JSON.stringify(lorebook, null, 2), 'utf-8')
  // 更新角色引用
  character.lorebookId = lorebookId
}
```

注意 `normalizeCharacter` 是 `async` 函数（因为 `downloadImageAsBase64`），所以同步写文件是安全的。

**无需修改前端 UI** ——世界书会自动出现在 LorebookPage 中。

### 修改 6：V1/V2/V3 适配确认与补全

**当前适配情况：**

| 版本 | 结构 | 导入 | 导出 | 状态 |
|------|------|------|------|------|
| V1 | flat JSON | `parsed.data ?? parsed` 回退到顶层 | V2 格式 | ✅ 支持 |
| V2 | `{spec, spec_version, data}` | `parsed.data` 读取 | V2 格式 | ✅ 支持 |
| V3 | 同 V2 data 包装 + 新字段 | `parsed.data ?? parsed` 兼容 | V2 格式 | ⚠️ 核心字段支持，缺少 V3 特有字段 |

**需要补充的 V2/V3 字段：**

在 `normalizeCharacter` 的参数解析和返回的 `Character` 接口中：

1. **`alternate_greetings`**（V2+）：备用问候语数组。可选：在首条消息中附加或单独存储。
2. **`creator_notes`**（V2+）：创作者备注。存入 `description` 末尾或新字段。
3. **`character_version`**（V3）：角色卡版本号。存储但不显示。

**修改方案：** 最小改动——不扩展 `Character` 接口，而是在导入时做兼容处理：

**文件：`electron/services/charCard.ts` - `normalizeCharacter` 函数**

```typescript
// 备用问候语处理：有 alternate_greetings 且无 first_mes 时使用第一个
if (!data.first_mes && !data.firstMessage) {
  const altGreetings = data.alternate_greetings
  if (Array.isArray(altGreetings) && altGreetings.length > 0) {
    data.first_mes = altGreetings[0] // 使用第一个备用问候语
  }
}
```

同时添加日志/注释表明支持 V1/V2/V3 三个版本。

## 假设与决策

1. **批量导入** - 使用 `multiSelections` 属性，一次对话框选择多个文件，逐个处理
2. **角色卡封面** - 采用竖版 3:4 比例，符合大多数角色卡画风
3. **长记忆总结** - 使用当前配置的 AI 提供商，总结请求使用独立 prompt，不影响对话上下文
4. **长记忆触发** - 自动模式在每次发送消息后检查消息数是否达到间隔倍数；手动模式通过按钮触发
5. **内嵌世界书导入** - 角色卡中的 `character_book` 数据提取为独立 `.json` 文件，自动关联
6. **V3 兼容** - 最小改动原则，仅补充 `alternate_greetings` 回退逻辑，不扩展 Character 类型
7. **视图模式** - 默认网格模式，用户可手动切换到列表模式；视图偏好不持久化（每次打开页面恢复默认网格）
8. **Persona 头像** - 使用与 CharacterEditor 相同的文件选择 + base64 读取流程，圆形裁剪展示

## 验证步骤

1. **批量导入**：多选 5 个 PNG/JSON 文件 -> 全部导入成功，显示"成功导入 5 个角色"
2. **批量导入含错误**：混入无效文件 -> 显示"成功导入 3 个角色，失败 2 个"，列出失败原因
3. **编辑页高度**：打开编辑器 -> description 和 firstMessage 文本框明显更高，支持拖拽调整
4. **封面展示**：角色卡网格中封面比例变为竖版 3:4，图片内容更多展现
5. **封面预览**：点击封面 -> 弹出大图 overlay，显示完整图片
6. **长记忆开关**：在会话中开启记忆 -> system prompt 中无记忆内容
7. **手动总结**：点击"立即总结" -> AI 生成摘要，存入 session，刷新后仍在
8. **自动总结**：设置每 10 条自动总结，对话 10 条 -> system prompt 中出现记忆内容
9. **统计信息**：面板显示对话次数、文字量、时长
10. **世界书自动导入**：导入含 `character_book` 的角色卡 -> LorebookPage 中出现对应世界书，角色 `lorebookId` 已关联
11. **V3 兼容**：导入含 `alternate_greetings` 但无 `first_mes` 的角色卡 -> 使用第一个备用问候语作为首条消息
12. **双模式切换**：角色管理页默认网格模式 -> 点击列表图标 -> 切换为横向列表布局，一行一张卡，左侧封面右侧信息 -> 再点网格图标恢复
13. **列表模式交互**：列表模式下操作按钮（聊天/编辑/导出/删除）始终可见，无需 hover
14. **Persona 头像**：身份编辑弹窗 -> 点击头像区 -> 选择图片 -> 头像显示在弹窗和卡片中 -> 保存后持久化
15. `npx tsc -b --noEmit` 零错误
