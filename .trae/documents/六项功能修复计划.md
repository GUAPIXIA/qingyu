# 六项功能修复与增强计划

## 概述

用户反馈 6 个问题：1) AI 回复需先输出心理描写再输出对话并区分显示；2) 对话中图片无法显示；3) 翻译文本无法复制；4) 单角色无法多对话；5) AI 一键翻译角色卡；6) JSON 导入角色卡无封面。

## 当前状态分析

| # | 问题 | 根因 |
|---|------|------|
| 1 | AI 回复无心理描写区分 | 系统提示词未要求 AI 分段输出；MessageBubble 无心理描写解析和样式 |
| 2 | 对话图片不显示 | 需调试 `file:readImageBase64` IPC 返回格式及 `<img>` 渲染 |
| 3 | 翻译文本无法复制 | 翻译 div 未加 `user-select: text`，继承 body 的 `user-select: none` |
| 4 | 单角色仅一个对话 | 存储为 `chats/{characterId}/messages.jsonl`，无 session 概念 |
| 5 | 无 AI 翻译角色卡功能 | CharacterEditor 无翻译按钮 |
| 6 | JSON 导入无封面 | `normalizeCharacter` 未检查 JSON 中的 `avatar`/`image`/`image_url` 等图片字段 |

## 修改计划

### 修改 1：AI 心理描写 + 对话区分显示

**原理：** 修改系统提示词，要求 AI 用 `<thought>...</thought>` 标签包裹心理描写，再输出正文。MessageBubble 解析此标签，将心理描写渲染为可折叠的样式区块。

**文件：** `src/store/useChatStore.ts` - `buildContext` 方法
- 在 systemContent 末尾追加指令：
  ```
  【输出格式要求】
  请先在 <thought>...</thought> 标签内输出角色的内心想法和心理活动，
  然后再输出角色的实际对话和行动。两部分必须分开。
  ```

**文件：** `src/components/chat/MessageBubble.tsx`
- 新增 `parseThought(content)` 函数，提取 `<thought>...</thought>` 内容
- 心理描写渲染为可折叠区块（默认折叠，点击展开）：
  ```tsx
  {thought && (
    <div className="mb-2 rounded-lg bg-tavern-bg-soft border border-tavern-border-soft px-3 py-2">
      <button onClick={toggleThought} className="text-xs text-tavern-text-muted">
        💭 内心想法 {expanded ? '▼' : '▶'}
      </button>
      {expanded && <div className="mt-1 text-sm italic text-tavern-text-muted select-text">{thought}</div>}
    </div>
  )}
  ```
- 正文部分移除 `<thought>` 标签后正常渲染

### 修改 6：JSON 导入角色卡自动提取封面

**原理：** JSON 角色卡中通常包含封面图片地址（`avatar`、`image`、`image_url` 等字段），当前 `normalizeCharacter` 完全忽略这些字段。修复后自动检测并下载/使用。

**文件：** `electron/services/charCard.ts` - `normalizeCharacter` 函数
- 新增图片字段检测逻辑，按优先级检查：
  ```typescript
  // 检查 JSON 中的图片字段（按常见字段名优先级）
  const imageUrl =
    data.avatar ?? data.image ?? data.image_url ??
    data.thumbnail ?? data.portrait ??
    parsed.avatar ?? parsed.image ?? parsed.image_url ??
    null
  ```
- 如果 `imageUrl` 是 `data:image/...;base64,...` 格式 -> 直接使用
- 如果 `imageUrl` 是 `http://` 或 `https://` URL -> 下载图片转为 base64
- 如果 `avatarBase64` 参数已传入（PNG 导入） -> 优先使用参数
- 如果都没有 -> 保持空字符串

**文件：** `electron/services/charCard.ts` - 新增 `downloadImageAsBase64(url)` 函数
- 使用 Node.js `https`/`http` 模块下载图片
- 转为 base64 data URL 返回
- 下载失败时返回空字符串（不阻断导入流程）
- 超时保护（10 秒）

**文件：** `electron/ipc/character.ts` - `importJson` handler
- 导入成功后检查 `character.avatar` 是否仍为空
- 如为空，在返回结果中标记 `needAvatar: true`

**文件：** `src/store/useCharacterStore.ts` - `importJson` 方法
- 检查返回结果，如 `needAvatar` 为 true 则自动打开 CharacterEditor

**文件：** `src/pages/CharactersPage.tsx`
- 导入 JSON 后如无头像，自动打开编辑器
- 编辑器中头像区域加提示文字"导入的角色卡未包含图片，请点击添加"

## 假设与决策

1. **心理描写格式** - 使用 `<thought>...</thought>` 标签，AI 输出时包裹内心想法，解析时提取并折叠显示
2. **多对话兼容** - 旧的 `messages.jsonl` 自动迁移为 `default.jsonl` 作为默认会话
3. **AI 翻译** - 逐字段翻译，使用当前配置的 AI 提供商，翻译时保留原始 Markdown 格式
4. **JSON 封面提取** - 自动检测 JSON 中的图片 URL 字段并下载；下载失败不阻断导入，后续引导手动添加
5. **图片下载** - 使用 Node.js 原生 http/https 模块，10 秒超时，失败时静默跳过
6. **不自动打包** - 按用户要求，修复后等测试确认再打包

## 验证步骤

1. AI 回复包含 `<thought>` 心理描写，折叠显示，正文正常
2. 用户发送图片，消息中图片正常显示
3. 翻译结果可自由选中和复制
4. 同一角色可创建多个对话，切换、删除正常
5. 角色编辑器中点击"AI 翻译"按钮，各字段被翻译为中文
6. 导入含 `avatar` URL 的 JSON 角色卡 -> 封面自动下载并显示
7. 导入不含图片字段的 JSON 角色卡 -> 自动打开编辑器提示添加头像
8. `npx tsc -b --noEmit` 零错误
9. `pnpm electron:dev` 无运行时错误
