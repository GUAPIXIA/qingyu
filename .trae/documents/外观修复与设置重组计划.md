# 外观设置修复 + API模型重构 + 设置页重组

## 概述

修复三项外观设置 Bug + 重构设置页结构 + 新增 TTS/识图/生图模型配置。

---

## Part 1: BUG 修复

### 1.1 消息间距无效

**根因**：
- `.spacing-normal` CSS 设 `padding: 8px`，与 `MessageBubble.tsx` 的 `py-2` (8px) 完全相同 → "标准"完全看不出差别
- `.spacing-compact` (4px) vs 默认 (8px) 只差 4px，极不显眼
- `.spacing-loose` (16px) vs 默认 (8px) 只差 8px，不明显
- 且用 `padding` 会被容器 `py-4` 稀释视觉效果

**修复**（`src/index.css` + `src/components/chat/MessageBubble.tsx`）：
- 改用 `margin-bottom` 控制消息间距，数值加大差距：紧凑 2px / 标准 12px / 宽松 24px
- MessageBubble 移除 `py-2`，改由 CSS 控制间距

### 1.2 主题模式（深色/浅色）不生效

**根因**：
- `tailwind.config.js` 中所有 `tavern` 颜色是静态硬编码暗色值（如 `bg: '#1a1625'`），没有浅色变体
- `index.css` 中 `body` 也是硬编码暗色 `background-color: #1a1625; color: #e8e0f0;`
- App.tsx 只给 `<html>` 加了 `dark`/`light` class，但没有任何 CSS 响应

**修复**（`tailwind.config.js` + `src/index.css`）：
- 将 `tavern` 颜色改用 CSS 变量
- 在 `:root`（浅色默认）定义亮色调，在 `:root.dark` 覆盖暗色调
- body 样式也用 CSS 变量

### 1.3 主题色变化不明显

**根因**：
- `.theme-amber` 等只设置 `--color-accent`，仅被 `btn-primary` 和 `tavern-accent` 类使用
- 大面积背景/文字/边框不使用 accent 色

**修复**（`src/index.css`）：
- 让 header 分割线、选中态、链接等更多元素使用 `border-tavern-accent` 和 `text-tavern-accent`，提升主题色存在感

### 1.4 字体大小（验证）

- App.tsx 逻辑正确：`<html>` 加 `font-compact/comfortable/loose` → 设 `--font-size-base` → `body { font-size: var(--font-size-base) }`
- 无需修改，保持现状

## Part 2: 设置页重组

### 2.1 新增全局模型字段（`shared/types.ts` + `src/utils/defaults.ts`）

```ts
// Settings 新增字段
ttsModel: string         // TTS 模型（如 tts-1）
visionModel: string      // 识图模型（如 gpt-4o）
imageGenModel: string    // 生图模型（如 dall-e-3）
```

默认值：空字符串，由用户自行填写。

### 2.2 设置页 Section 重组（`src/pages/SettingsPage.tsx`）

重组为 4 个清晰区域：

| 顺序 | Section | 内容 |
|------|---------|------|
| **1** | **API 连接** | 不变：Provider tabs + 快速配置 + Base URL + API Key + 聊天模型 + 测试连接 |
| **2** | **专项模型** | 新增：TTS 模型 / 识图模型 / 生图模型（全局，3 个独立输入框） |
| **3** | **外观设置** | 主题模式 + 主题色 + 字体大小 + 气泡样式 + 消息间距（修复后可即见效果） |
| **4** | **显示与行为** | 模型名快捷编辑 + 流式输出 + 自动滚动 + TTS 引擎（移入） |
| **5** | **数据管理** | 不变：导出/导入备份 |

原"模型与生成参数"Section 拆分为**专项模型** + **显示与行为**，语义更清晰。
