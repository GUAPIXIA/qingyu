# 酒馆(QingYu) vs SillyTavern-Launcher 项目差距分析报告

> 分析日期: 2026-07-21

---

## 一、项目定位总览

| 维度 | 酒馆 (QingYu) | SillyTavern-Launcher (STL) |
|------|:---:|:---:|
| **定位** | 独立桌面 AI 角色扮演客户端 | AI 生态系统全功能启动器 |
| **形态** | Electron 桌面应用 (带 GUI) | CLI 脚本菜单工具 (Batch/Shell) |
| **技术栈** | Electron 31 + React 18 + TypeScript + Vite 6 + Zustand | Windows Batch + Linux Shell (纯脚本, 无构建工具) |
| **支持平台** | Windows 10/11 x64 | Windows / Linux / macOS / Docker / Termux |
| **许可证** | 私有 | MIT |
| **版本** | 0.1.0 (早期) | 25.12.1.1 (成熟) |
| **包管理** | pnpm monorepo | 无 (纯脚本) |

## 二、架构级差距

### 2.1 核心技术路线

```
酒馆架构:
┌─────────────────────────────────┐
│  Electron 主进程                  │
│  ├── 本地存储 (文件系统)           │
│  ├── AI 调用 (OpenAI/Claude/...)  │
│  ├── MCP 工具管理                  │
│  ├── TTS 合成                     │
│  └── 凭据加密                     │
├─────────────────────────────────┤
│  IPC (contextBridge)             │
├─────────────────────────────────┤
│  渲染进程 (React + Vite)          │
│  ├── 对话页面                     │
│  ├── 角色管理                     │
│  ├── 世界书                       │
│  ├── 群聊                         │
│  ├── 预设管理                     │
│  ├── 正则规则                     │
│  └── 用量统计                     │
└─────────────────────────────────┘

STL 架构:
┌──────────────────────────────────┐
│  Launcher.bat / launcher.sh      │  ← 入口 (菜单驱动)
│  ├── 环境检测 (Git/Node/Python)   │
│  ├── 自更新 (git fetch)           │
│  └── 主菜单                       │
├──────────────────────────────────┤
│  Toolbox (工具箱)                 │
│  ├── App_Installer   安装管理     │
│  ├── App_Launcher    启动管理     │
│  ├── App_Uninstaller 卸载管理     │
│  ├── Backup          备份恢复     │
│  ├── Editor          配置编辑     │
│  └── Troubleshooting 故障排除     │
├──────────────────────────────────┤
│  SillyTavern/ (主应用)            │  ← Git clone 的完整应用
│  text-completion/ (LLM 后端)      │
│  image-generation/ (生图后端)      │
│  voice-generation/ (语音后端)      │
└──────────────────────────────────┘
```

### 🔴 关键差距: STL 是"生态管理器", 酒馆是"单体应用"

- STL 管理着 **SillyTavern 主应用 + 20+ 个 AI 生态工具** 的完整生命周期
- 酒馆只是 **一个独立的 Electron 应用**, 没有管理外部生态的能力
- STL 通过启动器间接运行 SillyTavern (Node.js Express 服务), 酒馆自己就是客户端

## 六、总结: 核心差距矩阵

### 🔴 严重缺失 (P0)

1. **本地 LLM 后端支持** — 无法对接 Oobabooga/KoboldCPP/LlamaCPP/TabbyAPI 等本地推理引擎
2. **本地生图后端支持** — 无法对接 SD WebUI/ComfyUI/Forge 等本地生图引擎
3. **本地语音后端支持** — 无法对接 XTTS/AllTalk/RVC 等本地语音引擎
4. **跨平台支持** — 仅 Windows, 缺少 Linux/macOS/Docker
5. **应用更新机制** — 无自动更新能力

### 🟡 中度缺失 (P1)

6. **插件/扩展系统** — 无扩展机制, 功能无法由社区扩展
7. **向量化记忆** — 无长期记忆/向量数据库
8. **备份系统** — 仅有设置导入导出, 无完整数据备份恢复
9. **GPU/VRAM 检测** — 无法检测硬件并推荐模型
10. **新手引导** — 无首次使用引导流程
11. **角色标签/市场** — 缺少标签系统和角色市场集成

### 🟢 轻微差距 (P2)

12. **SSL 管理** — 无证书生成
13. **远程访问** — 无 Cloudflare Tunnel
14. **系统诊断** — 无系统级故障排除工具
15. **CI/CD** — 无自动化流水线
16. **多语言 UI** — 无国际化框架
17. **角色卡标签** — 缺少角色分类标签

### ✅ 酒馆的优势

相比 STL, 酒馆有以下独特优势:

1. **原生桌面体验** — Electron 原生窗口, 非浏览器 tab, 更沉浸
2. **现代 UI** — React + Tailwind + 暗色主题 + 6 种主题色, 视觉效果远胜 CLI
3. **凭据安全加密** — Electron safeStorage 加密存储 API Key
4. **6 主题色 x 3 字体 x 3 气泡** — 深度个性化定制
5. **MCP 协议支持** — 内置 MCP 工具管理 (AI 可调用外部工具)
6. **用量统计** — Token 用量 + 费用计算 + 可视化
7. **一体化** — 安装即用, 无需管理多个外部依赖
